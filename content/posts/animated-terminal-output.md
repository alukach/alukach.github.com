Have you just written a new ‚ú®fancy CLI‚ú® and want to demo it in your Github Readme? Recording your terminal output is a nice way to demonstrate the experience.

Here's an example of what we're going to make:

![](https://gist.githubusercontent.com/alukach/7c510d45080d5b2c1d42a0309ad25411/raw/9e42a8904b8da9a289d45b44491f39a1586293fb/example.svg)


## Steps

### Install Dependencies

1. [asciinema](https://asciinema.org/): `brew install asciinema`
1. [svg-term-cli](https://github.com/marionebl/svg-term-cli): `npm install -g svg-term-cli`

### Setup your terminal

Some tips:

1. Font/screen size matters. The asciinema output will look just as it does in your terminal.  You'll probably want to bump up the font-size and shrink down the terminal so that the text is legible in your README.
2. Choose a nice, generic terminal theme.  If using [ohmyzsh](https://ohmyz.sh/), I recommend the [`arrow` theme](https://github.com/ohmyzsh/ohmyzsh/wiki/Themes#arrow).

### Lights, camera, action üé¨

The `asciinema rec` command will start a process to capture the stdout of your terminal session.  When you're done, pressing <kbd>ctrl</kbd>+<kbd>d</kbd> will end recording.  You can then opt to save it to the asciinema server or locally.  For our needs, we'll save locally.  

### Post Production ‚úÇÔ∏èüéû

asciinema will place the file in a temporary directory.  The output "cast" is a [simple newline-delimeted JSON file](https://discourse.asciinema.org/t/can-i-edit-post-process-the-recording/25), which a savvy user could edit by hand to scrub out any mistakes or personal information.

### Transform to SVG

Now, for us to place a cast into a Github README or Issue, we need to convert the cast to an animated SVG.  [svg-term-cli](https://github.com/marionebl/svg-term-cli) allows us to do this with either a raw cast file or the ID of a cast uploaded to asciinema.  Adding the `--window` flag wraps the cast in a MacOS-style terminal window.

```sh
svg-term --in /var/folders/3n/3dct2kb54xn1882dpbw0vbdh0000gn/T/tmp0m0d4ri1-ascii.cast --out /tmp/example.svg --window
```

### Embedding

Now that we have our animated SVG, we just need to put it online and reference it.

#### README

If you're looking to add your SVG to a README, you can include the SVG in the git repo (e.g. in `docs/`) and reference it in markdown:

```md
![example cli usage](./docs/example.svg)
```

#### Issue

Unfortunately, Github does not allow users to drag/drop SVGs into Github Issues.

<img width="856" alt="Screen Shot 2021-09-24 at 1 16 53 PM" src="https://user-images.githubusercontent.com/897290/134733055-b250972a-52de-4e7c-8a19-6898f486f3ac.png">

Instead, you must figure out your own way to upload the SVG to the internet. One simple solution is to upload it to a [gist](https://gist.github.com) and then reference the images from your Github Issue.

```md
![](https://gist.githubusercontent.com/alukach/7c510d45080d5b2c1d42a0309ad25411/raw/9e42a8904b8da9a289d45b44491f39a1586293fb/example.svg)
```
